# Cloudflare Worker API 프록시 설정 가이드

이 가이드는 NEIS API 키를 숨기고 보안을 강화하기 위한 Cloudflare Worker 설정 방법을 안내합니다.

## 1. Cloudflare Worker 코드 (index.js)
아래 코드를 복사하여 Cloudflare Worker 편집기에 붙여넣으세요.

```javascript
export default {
  async fetch(request, env) {
    // 1. 허용할 도메인 설정 (내 웹사이트 주소)
    const ALLOWED_ORIGINS = [
      "https://schoolmealproject-auhvl.run.goorm.io", 
      "http://localhost:3000"
    ];

    const origin = request.headers.get("Origin");
    
    // CORS 헤더 설정
    const corsHeaders = {
      "Access-Control-Allow-Origin": ALLOWED_ORIGINS.includes(origin) ? origin : ALLOWED_ORIGINS[0],
      "Access-Control-Allow-Methods": "GET, OPTIONS",
      "Access-Control-Allow-Headers": "Content-Type",
    };

    // 브라우저 사전 검사(OPTIONS) 처리
    if (request.method === "OPTIONS") {
      return new Response(null, { headers: corsHeaders });
    }

    // URL에서 날짜 파라미터 추출
    const { searchParams } = new URL(request.url);
    const apiDate = searchParams.get("date");

    if (!apiDate) {
      return new Response(JSON.stringify({ error: "date 파라미터가 누락되었습니다." }), {
        status: 400,
        headers: { ...corsHeaders, "Content-Type": "application/json" }
      });
    }

    // NEIS API 호출 (환경변수 NEIS_API_KEY 사용)
    const API_EDUCODE = "Q10";
    const API_SCHOOLCODE = "8490325";
    const neisUrl = `https://open.neis.go.kr/hub/mealServiceDietInfo?KEY=${env.NEIS_API_KEY}&Type=json&ATPT_OFCDC_SC_CODE=${API_EDUCODE}&SD_SCHUL_CODE=${API_SCHOOLCODE}&MLSV_YMD=${apiDate}`;

    try {
      const response = await fetch(neisUrl);
      const data = await response.json();
      
      return new Response(JSON.stringify(data), {
        headers: { ...corsHeaders, "Content-Type": "application/json" }
      });
    } catch (error) {
      return new Response(JSON.stringify({ error: "데이터를 불러오는 중 오류가 발생했습니다." }), {
        status: 500,
        headers: { ...corsHeaders, "Content-Type": "application/json" }
      });
    }
  }
};
```

## 2. API 키 설정 (중요!)
배포(Deploy) 후 다음 단계를 반드시 수행하세요:
1. Cloudflare Worker 상세 페이지 진입
2. [Settings] -> [Variables] 클릭
3. [Environment Variables] 섹션에서 [Add variable] 클릭
4. Variable name: 
5. Value: 실제 API 키 입력
6. [Encrypt] 버튼 클릭하여 암호화 (선택 사항이지만 권장)
7. [Save and deploy] 클릭

## 3. 프론트엔드 연동
설정이 완료되면 Worker의 주소(예: https://project.name.workers.dev)를 저에게 알려주세요. 
`getSchoolInfo.js` 파일을 수정하여 해당 주소를 사용하도록 업데이트해 드리겠습니다.
